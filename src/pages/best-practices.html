Focus management

According to 2.4.3 Focus Order we must provide our users with a predictable focus order and focus indicator movement throughout all of their interactions with our application.

Clearable inputs

Once we press on the "Clear" icon button, we need to move the focus back to the corresponding input.

Deletable options

When we have multiple deletable option items in a list - deleting an item should move the focus to the next appropriate logical place on the page.

  1. If we are deleting the last element in the list - the focus should move to the item before it (second to last element)
  2. If we are deleting any other element (N) - the focus should move to the item after it (N+1)

Tooltips

Tooltips are one of the most widely used pieces of UI out there and they are usually used incorrectly. They should be used to either label an interactive element (ex. button, input, icon button etc.) or describe it.

When we use Tooltips as helpful additional descriptions, we need to of course programmatically connect it with the button using aria-describedby.

Toggletips

Similar to Tooltips, Toggletips toggle their content on the "click" event.
They are usually presented as an "info" icon button whose sole purpose is to toggle the content of the Toggletip.
