Animation & Reduced motion

Animations can be quite distracting to people that suffer from vertigo, have ADHD or just have a high sensitivity to motion.
As such, we need to make sure we minimize (or completely remove) distracting animations whenever possible.

A good way to make sure we are are not distracting our users is to respect their Reduced Motion preferences.

(Missing) Focus Indicators

Any interactive element must have a focus indicator to let (keyboard-only) users know where they are on the page.

This doesn't mean that we MUST use the outline to do so.

Custom buttons

We always want to prefer native implementations over custom ones if we can help it and nowhere is this more easily prevalent than in good ol' buttons.


A native button gives us all necessary and accessible features we expect - out of the box:

  • Proper semantic role: "button"
  • Accessible keyboard controls - Enter & Space keys activate the "click" event
  • Clean focus management - the button is correctly placed in the focus order of the page
  • Clean HTML attributes for connecting it with forms, disabling it etc.
